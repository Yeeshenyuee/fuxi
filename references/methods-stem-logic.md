# A 类方法:理工逻辑·推导计算

适用:高数、物理、电路、力学、算法、化学原理、统计…… 这类的命门是**想通底层逻辑**,公式靠理解记,题靠套路 + 练。

## 讲解四件套(每个考点)

1. **底层逻辑**:第一性原理 + 大白话 + 至少一个生活类比(见下"造类比")。回答"这东西本质在干嘛、为什么这样"。
2. **机制图 / 推导图**:一张 SVG(机制图、特性曲线、框图、推导流程)。一图一核心。
3. **公式卡**:每个公式给"**来历(怎么推出来的)+ 每个符号含义 + 什么时候用 + 易错点**",绝不让用户死背符号。
4. **题型 + 计算专练**:把该章题归成"题型套路"(看到X用Y);计算题逐步代数字 + **求和/量纲/代回自检** + 换数练习。

## 公式怎么教(关键)

- 先问"这公式想算什么",再看它怎么来的(推导或物理意义),最后才是套用。
- 相似公式用**对比表**并排(易混的 R\*、KL、KG 这类)。
- 标"**陷阱**":如净缺额=损失−备用、符号正负、标幺与有名值换算。

## 题型套路库(每章建)

- 列"看到什么关键词 → 用哪个公式/方法 → 第几步"。让用户**先认题型**再动手。
- 错题归因:错在"概念不清 / 套路没认出 / 计算失误 / 陷阱踩坑"哪一类,对症补。

## 造类比的通法(类比是 A 类利器)

- 把抽象量映射到日常经验(频率→转速/体温;无功→撑场子的钱;调差系数→弹簧软硬)。
- 类比要能**延展**到该概念多个性质,不止一个点;讲完**映射回术语**。
- 验收:撤掉类比、只看术语,用户还懂吗?要懂才算成功。

## 推荐图型(详见 diagram-recipes.md)

机制图、特性曲线/坐标图、接线/结构框图、波形/周期图、推导流程、题型决策树。

## 自测方式

以**计算题(换数)**为主,配概念辨析、推导题、读图/画图题。批改时查每一步,指出错在哪步、为什么。
