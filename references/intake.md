# 开课前 intake(资料确认 + 学习者画像)

正式复习前必须完成三件事,**别跳过**——对齐了资料和学习者情况,后面才高效。"磨刀不误砍柴工"。

**铁律:用户一说复习,先做 intake 提问 → 停下等回答;资料由用户提供,绝不自己联网搜。**

## 一、资料确认与管理(先确认有什么资料,才开始正式复习)

维护一份 `资料清单.md`(同时可映射到 `进度.json` 的 `materials` 数组)。每份资料记:名称、类型、来源/路径、覆盖范围、状态(active / removed)。

- **开始前必确认**:问用户"**这些就是全部资料吗?还要加吗?**"——确认齐全,再进阶段1抽料、阶段2建图谱。
- **随时可改(用户任何时候提出都响应)**:
  - **加资料**:追加进清单 → 抽料 → 更新知识地图与受影响讲解。
  - **删资料**:状态标 removed → 后续讲解不再依赖它;已生成内容里引用到它的地方,提示用户可能需重讲。
  - **换/改资料**:替换版本 → 重新抽料 → 更新受影响章节。
  - **看清单**:随时展示当前 `资料清单.md`。
- **隐私提醒(收资料时说一句)**:`复习/<科目>/` 里是用户的版权教材与个人学习数据,**别推到公开仓库或对外分享**;fuxi 技能仓库与学习数据目录是分开的,推送技能更新不会带上学习数据。
- **没有任何资料时**:请用户补发;**不要自己联网搜索复习资料**。只有当用户明确说"我没有,你来定"时,才可用你已有的**标准知识**讲(并说明"这不基于你的教材");如需联网核实,必须**先征得用户同意并注明来源**。

## 二、学习者画像(问清基础 + 偏好,写进 `进度.json` 的 `learner_profile`)

开课前做个 **30 秒小访谈**。**把下面这份「菜单式模板」原样发给用户**(带字母选项),用户回字母即可——无按钮环境(Codex / 终端)也能一键选、不用打整句;有选择 UI(如 Claude 的 AskUserQuestion)时可改用之。

```
先花 30 秒对齐,直接回编号+字母即可(例:2A 3A 4A+B 5A):

1. 资料:把教材 / 课件 / 真题 / 作业 / 老师划的重点(或截图)发我;没有就回「1无」。
2. 基础:A 零基础   B 学过忘了   C 有基础想冲刺
3. 目的:A 考试·求及格   B 考试·冲高分   C 自学 / 兴趣   (考试的话补一句日期,如「7月10日考」→ 我会生成逐日倒计时计划)
4. 偏好(可多选):A 多配图   B 多例题   C 口诀记忆   D 先框架后细节   E 快速过划重点
5. 节奏:A 小步(一小节一停)   B 一次多讲点
6. 想法(选填):最怕啥 / 想侧重啥 / 老师强调过啥,一句话也行

资料给了 + 上面选好,我就开始。
```

**把用户回的字母映射进 `learner_profile`**:2→基础,3→目的/目标,4→偏好(可多选),5→节奏,6→诉求(自由文本,优先级最高)。用户没答的项按默认(零基础 / 多图+多例题 / 小步),别卡着不放。

## 三、画像 → 讲解策略(据此调详略,这是"更高效"的关键)

| 画像 | 讲解怎么调 |
|---|---|
| 零基础 | 更多大白话+类比+图,步子更小,先讲最核心主线 |
| 有基础 / 冲刺 | 跳过基础,直击难点、易错、押题,节奏加快 |
| 时间紧 | 优先押题 + 核心考点 + 速记卡,砍掉枝节 |
| 偏图 | 每个点都配图,少堆文字 |
| 偏例题 | 多用例子/分录/计算演练带出概念 |
| 偏记忆 | 多上口诀、对比表、间隔重复 |
| 偏框架 | 先脑图铺全局,再逐块填肉 |
| 想快速过 | 只讲重点 + 押题,详解按需展开 |

**用户的自述诉求(`诉求` 字段)优先级最高**——直接照它调整(如"只突击重点"就砍全面讲、直奔押题;"最怕分录"就在该处多配图多例题)。

学习过程中用户改主意(如"我其实有点底子 / 想快点 / 图太多了")→ **随时更新 `learner_profile` 并即时调整**讲法。
