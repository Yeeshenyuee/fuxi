# 开课前 intake(资料确认 + 学习者画像)

正式复习前必须完成三件事,**别跳过**——对齐了资料和学习者情况,后面才高效。"磨刀不误砍柴工"。

## 一、资料确认与管理(先确认有什么资料,才开始正式复习)

维护一份 `资料清单.md`(同时可映射到 `进度.json` 的 `materials` 数组)。每份资料记:名称、类型、来源/路径、覆盖范围、状态(active / removed)。

- **开始前必确认**:问用户"**这些就是全部资料吗?还要加吗?**"——确认齐全,再进阶段1抽料、阶段2建图谱。
- **随时可改(用户任何时候提出都响应)**:
  - **加资料**:追加进清单 → 抽料 → 更新知识地图与受影响讲解。
  - **删资料**:状态标 removed → 后续讲解不再依赖它;已生成内容里引用到它的地方,提示用户可能需重讲。
  - **换/改资料**:替换版本 → 重新抽料 → 更新受影响章节。
  - **看清单**:随时展示当前 `资料清单.md`。
- **没有任何资料时**:告诉用户两条路——(a)发资料(PDF/图片/docx/PPT/md),或(b)我按该科**标准内容 + 联网核实**来讲(务必注明来源)。让用户选,别默默硬讲。

## 二、学习者画像(问清基础 + 偏好,写进 `进度.json` 的 `learner_profile`)

开课前做个 **30 秒小访谈**(一次性把问题抛给用户,别逐条盘问,允许"不确定"):

1. **基础**:这门课你现在什么程度?——零基础 / 学过忘了 / 有基础想冲刺。
2. **目的与时间**:考试还是自学?考试哪天(紧不紧)?目标及格还是高分?
3. **偏好**(可多选):更吃哪种讲法?——多配图直观 / 多例题演练 / 口诀记忆 / 先框架后细节 / 快速过划重点。
4. **节奏**:一次讲一小节就停下来问,还是一次多讲点?
5. **你自己的想法(开放,最重要)**:你想怎么开始、侧重什么、有什么特别要求或痛点?——如"最怕分录""只想突击考试重点""多举生活例子""按老师划的重点来""先做题再讲"。这条自由发挥,写进 `learner_profile.诉求`,常比选项更能定制学习。

## 三、画像 → 讲解策略(据此调详略,这是"更高效"的关键)

| 画像 | 讲解怎么调 |
|---|---|
| 零基础 | 更多大白话+类比+图,步子更小,先讲最核心主线 |
| 有基础 / 冲刺 | 跳过基础,直击难点、易错、押题,节奏加快 |
| 时间紧 | 优先押题 + 核心考点 + 速记卡,砍掉枝节 |
| 偏图 | 每个点都配图,少堆文字 |
| 偏例题 | 多用例子/分录/计算演练带出概念 |
| 偏记忆 | 多上口诀、对比表、间隔重复 |
| 偏框架 | 先脑图铺全局,再逐块填肉 |
| 想快速过 | 只讲重点 + 押题,详解按需展开 |

**用户的自述诉求(`诉求` 字段)优先级最高**——直接照它调整(如"只突击重点"就砍全面讲、直奔押题;"最怕分录"就在该处多配图多例题)。

学习过程中用户改主意(如"我其实有点底子 / 想快点 / 图太多了")→ **随时更新 `learner_profile` 并即时调整**讲法。
