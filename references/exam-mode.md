# 自测模式(出卷 + 批改)

> **先看类型再选自测方式**:有明确考试题型(`进度.json` 的 `exam_format`)就按它出卷;没有就按学科类型默认——A 计算/推导/辨析、B 默写/名词解释/合书主动回忆、C 听写/完形/翻译/写作、D 案例/操作步骤复述。各类细则见对应 `methods-*.md` 的"自测方式"。

## 出卷

1. 读 `进度.json` 的 `exam_format`(如 2计算+2分析+5简答+10填空),**严格按题型与数量出**。
2. 覆盖面:从 `知识地图.md` 各章考点均衡取材,优先押重点(计算题、画图/分析题、高频简答)。向薄弱章节适当加码。
3. **新数字**:计算题尽量换成与原作业不同的数字(真测水平),但自己先把正确答案算出来留作批改用;选能算出较整齐结果的数。
4. **先只发题、不发答案**,让用户真做。说明:可拍照/打字,可只做一部分。画图题让其在纸上画后拍照。
5. 配分像真卷(如 填空2分×10、简答6分×5、分析12分×2、计算13分×2 = 100),给点临场感。

## 批改

1. 用户交多少批多少。逐题:**对/错/部分对** + 给正确答案/满分要点。
2. 错题:**别只给答案**,回到阶段3用更简单的方式重讲该点(换类比/换图/拆步骤),并更新 `进度.json` 的 `weak_points`。
3. **每道错题写进 `错题本.md`**(题目/用户答案/错因分类/正解要点/重练日期;模板见 `templates/错题本.md`,错因→补法映射见 `references/memory-tools.md`)。重练=同类题换数字再出,做对才标"已过关"。
4. 计算题批改:检查每一步(尤其易错点),指出错在哪一步、为什么。
5. 给一个简短总评 + "下一步建议练哪类"。
6. **按 `references/memory-tools.md` 的判定规则更新各章掌握度**(mastered / 薄弱),写回 `进度.json`。

## 出卷取材加权
出新卷时,**优先覆盖 `weak_points` 与错题本里"待重练"的同类考点**(换数字/换情境),其余名额再从知识地图均衡取——测该补的,不是重复测已会的。

## 题型答法要求(写参考答案时)

- **填空**:给标准词,必要时一句话点为什么。
- **简答**:给"满分答法模板"(分点、踩点)。
- **计算**:给公式来历 + 逐步 + 自检。
- **分析/画图**:给"判断步骤 + 要画的关键结构 + 结论句",画图题描述清楚画什么、标什么。
