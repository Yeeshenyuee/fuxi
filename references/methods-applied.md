# D 类方法:应用·操作·实务

适用:编程实操、会计实务、临床/实验操作、工程规范、各科"案例分析"题。命门是**流程 + 决策 + 动手 + 规范**,光看不练等于没学。

## 讲解重点

1. **流程图**:把操作拆成有序步骤(谁→做什么→产出什么)。
2. **决策树**:遇到不同情况怎么选(if 这样 then 那样)——实务题多是判断题。
3. **checklist**:关键步骤/易漏点列成清单,操作时逐项核对。
4. **常见坑**:每个流程标"新手常错在哪、为什么"。

## 案例走查(case walkthrough)

拿一个典型案例,**带着用户从头走一遍**:识别情形→选方法→执行步骤→得结论。再换一个变体让他自己走。

## 动手优先

能动手的让用户动手(写代码、做账、画流程),而不是只听讲。本 skill 给"模拟题/案例"让其练。

## 推荐图型

流程图、决策树、结构/架构图、checklist 表。

## 自测方式

案例分析题、操作步骤复述(合上书说出完整流程)、改错(给个有坑的流程让他找错)、限时实操。
