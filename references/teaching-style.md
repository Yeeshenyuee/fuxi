# 讲解策略总纲(先分类,再选法)

不同学科的"学法"根本不同:理工靠**想通逻辑**,文科靠**高效记忆**,语言靠**积累+输出**,实务靠**流程+动手**。所以本 skill 的第一原则是:**先判类型,再套对应方法**,绝不用一套讲法套所有科目。

## 一、普适原则(任何学科都适用)

1. **懂 > 背**:凡是能理解的,先讲透"为什么"——理解能大幅减少要背的量;**确实要硬背的(年份、条文、词汇),才上记忆术,而不是死记**。
2. **图文并重**:每个关键点配最合适的图(理工→机制图/曲线/框图;文科→思维导图/时间线/对比表/因果链)。图型见 `diagram-recipes.md`。
3. **主动回忆 + 间隔重复(记忆科学核心,所有学科都灵)**:学完立刻**合上资料**复述/默写/做题;标记易忘点,**隔天、隔3天、隔1周**回看。被动重读效果差,主动检索才记得牢。
4. **大白话**:先一句话直觉,再上术语;符号/概念即时翻译;⚠️ 标易错点;讲完能"撤掉脚手架still 懂"才算成功。
5. **按需逐章 + 进度续学**:跟着用户节奏,进度/薄弱点存盘。

## 二、第一步:给学科 / 考点分类(决定用哪套方法)

**学科很少是纯一类,按考点逐个打标签更准**(在阶段2建知识图谱时做)。五类:

| 类型 | 典型学科 | 核心需求 | 方法文件 |
|---|---|---|---|
| **A 理工逻辑·推导计算** | 高数、物理、电路、力学、算法、化学原理、统计 | 想通原理 + 推导 + 题型套路 + 计算 | `methods-stem-logic.md` |
| **B 文科记忆·框架概念** | 历史、政治、马原/思修、人文地理、法理、文学/管理常识 | 框架化 + 高效记忆 + 考场精准提取 | `methods-humanities-memory.md` |
| **C 语言** | 英语、二外、古文、语文字词 | 词汇 + 语法 + 用法 + 听说读写输出 | `methods-language.md` |
| **D 应用·操作·实务** | 编程实操、会计实务、临床/实验操作、工程规范、案例分析 | 流程 + 决策 + 动手 + 规范 | `methods-applied.md` |
| **E 混合** | 经济学、医学、法律、生物、地理 | 拆成上面几类,**按每个考点的类型分别套** | 按点取对应文件 |

**判型只问一句**:这个考点主要是要"**算/推**"(A)、"**记/背**"(B)、"**会用语言**"(C)、还是"**按步骤做**"(D)?
- 例:经济学的"供需曲线推导"是 A,"学派与代表人物"是 B;医学的"心脏传导机制"是 A,"药名剂量"是 B。同一科里混着判。

判完类型,读对应 `methods-*.md` 取该类的讲法、图型、记忆/练习手段、自测方式,再生成讲解。

## 三、通用技巧(各类都可借用)

- **类比**(尤其 A 类):把抽象映射到日常经验,讲完**映射回术语**。具体怎么造见 stem-logic 文件。
- **框架先行**(尤其 B 类):先给骨架(分类树/时间线/因果链)再填细节。
- **对比表**:任何易混点都用并排对比表(A 的相似公式、B 的易混概念、C 的近义词)。
- **主动回忆**:每讲完一块,让用户合上资料**自己复述一遍**再继续——这是检验"真懂/真记住"的金标准。
